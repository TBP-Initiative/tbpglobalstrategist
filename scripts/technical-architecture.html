1. System Architecture Overview

TBP Global Strategist is a full-stack web application built on Next.js 16 with the App Router pattern. The system follows a monolithic architecture with server-side rendering (SSR), API routes, and static generation where appropriate.

5. Authentication & Authorization

5.1 Authentication Flow

Credentials Provider:
1. User submits email + password via login form
2. NextAuth validates credentials against User table
3. Password verified using bcryptjs
4. JWT session token created and stored in cookie
5. User redirected to dashboard
Google Provider:
1. User clicks "Continue with Google"
2. Redirected to Google OAuth consent screen
3. Google returns authorization code
4. NextAuth exchanges code for tokens
5. User profile created/linked via PrismaAdapter
6. JWT session token created

5.2 Authorization Rules

Route PatternRequirement
/dashboard/*Authenticated (any role)
/dashboard/admin/*Authenticated + ADMIN role
/api/profile/*Authenticated
/api/projects (POST)ADMIN role
/api/admin/*ADMIN role
/api/projects (GET)Public
/api/strategistsPublic
/login, /registerPublic (redirect if already authenticated)

5.3 Session Strategy

  • Strategy: JWT (JSON Web Tokens)
  • Storage: HTTP-only cookies
  • Adapter: PrismaAdapter for session/account storage in database
  • Trust Host: Enabled for Vercel deployment

6.3 File Upload Flow

1. Client sends file via multipart form to /api/upload
2. Server validates file type and size
3. File uploaded to Supabase Storage bucket
4. Public URL returned to client
5. URL stored in database (ProjectMedia, Submission, etc.)

10. Security Considerations

10.1 Authentication Security

  • Passwords hashed with bcryptjs (salt rounds: 12)
  • JWT tokens stored in HTTP-only cookies
  • Session invalidation on logout
  • CSRF protection via NextAuth

10.2 API Security

  • Route protection via NextAuth middleware
  • Role-based access control (RBAC) for admin routes
  • Input validation with Zod schemas
  • SQL injection prevention via Prisma ORM

10.3 Data Security

  • Environment variables for sensitive configuration
  • Supabase RLS (Row Level Security) policies
  • No secrets committed to git repository
  • .gitignore excludes .env, node_modules, .next

10.4 Frontend Security

  • Content Security Policy headers (via Vercel)
  • XSS prevention via React's automatic escaping
  • HTTPS enforced in production
  • Secure cookie attributes (HttpOnly, Secure, SameSite)

11. Performance & Optimization

11.1 Rendering Strategy

StrategyUsage
Static Generation (SSG)Marketing pages, public profiles, project listings
Server-Side Rendering (SSR)Dashboard pages, admin panels, dynamic content
Client-Side RenderingInteractive components, real-time features, modals

11.2 Optimization Techniques

  • Code Splitting: Automatic with Next.js App Router
  • Image Optimization: Custom image-optimize.ts for client-side optimization
  • Tailwind Purging: Automatic unused CSS removal
  • React Query Caching: Stale-while-revalidate strategy for API data
  • Bundle Analysis: Next.js built-in bundle analyzer

11.3 Database Optimization

  • Connection Pooling: PgBouncer for efficient connection management
  • Indexed Queries: Prisma indexes on frequently queried fields
  • Selective Fields: Prisma select for fetching only needed data
  • Pagination: Cursor-based and offset pagination for large datasets
